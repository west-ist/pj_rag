必看，照着执行即可！！！（部分说明由llm辅助生成）
确保电脑已安装：
Python 3.8 或更高版本（在终端输入 python --version 检查）
pip

打开终端，切换到项目文件夹：
cd 你的项目路径/pjt_rag

创建虚拟环境（推荐，但不是必须）
# 创建虚拟环境（venv是环境名称，可自定义）
python -m venv venv
# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate
激活后，终端行首会出现 (venv) 字样。


安装所有依赖
确保终端路径在项目根目录（包含 requirements.txt 文件），执行：
pip install -r requirements.txt


配置 API Key（千问）
在项目根目录下找到 .env.example 文件。
复制一份并重命名为 .env（注意前面有个点）。
Windows 可能提示“必须输入文件名”，可以用命令：copy .env.example .env
或直接新建文本文件，命名为 .env
用记事本/VS Code 打开 .env，将 your-actual-api-key 替换为真实的千问 API Key

为防止盗刷，请大家自行申请API Key，免费额度肯定够用
#申请网址#：https://bailian.console.aliyun.com
在阿里云https://www.aliyun.com完成注册和实名认证后，在#申请网址#的常用功能中选择API Key，创建一个即可。

！！运行主程序！！
python main.py
（这里初次评估的速度会比较慢，耐心等待TAT）

负责测试评估时必看！！！
1.
在运行后打开outputs/qa_eval.csv
找到correct列（当前为空）。
对照answer标准答案，判断rag_answer是否正确：
正确 → 填1
错误 → 填0
填的时候注意不要把文件编码改了，要不然失败分析会报错
2.
再次运行主程序或单独运行failure_analyzer（建议单独运行，主程序耗时较长）
此时会在outputs中生成失败案例的文档，可据此人工进行分析失败原因
3.
单独运行visualize.py，在outputs中生成两种回答的准确率对比图
4.
进行三种chunk大小的对比实验，单独运行chunk_experiment.py，手动标注correct后，按指令生成可视化对比图

注：
1.如果对知识库或代码进行更新后，要重复运行，需要在运行前先删除旧的embedding缓存文件
执行：rm -f outputs/embeddings_cache.pkl
Windows下执行：del outputs\embeddings_cache.pkl

2.如果很长时间卡住不动，可以ctrl+c终止当前运行，然后重来。
